# 🕵️ Fraud Detection Analysis

This project analyzes insurance claim data to identify fraudulent patterns and help organizations reduce financial losses through data-driven decision-making.

---

## 🏢 Business Problem

Insurance companies process thousands of claims, but a significant portion are fraudulent, leading to major financial losses.

Manual fraud detection is:
- Time-consuming  
- Inefficient  
- Often too late  

This project aims to **predict fraudulent claims early** and improve decision-making.

---

## 🎯 Objective

To analyze historical claim data and build a system that can:
- Identify fraud patterns  
- Classify claims as fraudulent or legitimate  
- Support faster and more accurate claim approvals  

---

## 📊 Dataset Overview

The dataset includes:
- Customer demographics  
- Claim details (amount, type, severity)  
- Incident-related information  
- Policy and history data  

---

## 🔍 Analysis Approach

- Data cleaning and preprocessing  
- Feature engineering and variable transformation  
- Exploratory Data Analysis (EDA)  
- Correlation and statistical analysis  
- Model building using classification techniques  

---

## 🤖 Models Used

- Logistic Regression  
- Random Forest  

Both models were evaluated using:
- Accuracy  
- Precision & Recall  
- Sensitivity & Specificity  

---

## 📈 Key Insights

- Certain customer behavior patterns are strongly linked to fraudulent claims  
- Claim severity plays a significant role in fraud likelihood  
- Some customer segments show higher fraud probability  
- Fraud can be predicted early using historical data  

---

## 💡 Business Recommendations

- Increase verification for high-risk customer segments  
- Prioritize investigation for high-risk claim types  
- Reduce manual effort by using predictive scoring  
- Improve fraud detection efficiency early in the process
